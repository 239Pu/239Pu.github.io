发表于 2019-10-26 分类于 环境搭建

依靠github搭建hexo博客

开始前的准备

注册github账号
安装node 与 git
注册完github账号登录 新建仓库
设名称为你注册的用户名+githuo.io**


配置完成后不会立即生效可能会等10分钟左右

配置ssh key

cd ~/. ssh #检查本机已存在的ssh密钥

如果提示：No such file or directory 说明你是第一次使用git。

ssh-keygen -t rsa -C "邮件地址"

然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，
找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，
打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：

测试是否成功CMD输入

ssh -T git@github.com # 注意邮箱地址不用改

看到如下字样表示成功连接
You've successfully authenticated, but GitHub does not provide shell access.

git config --global user.name "liuxianan"// 你的github用户名，非昵称
git config --global user.email  "xxx@qq.com"// 填写你的github注册邮箱

安装Hexo

安装完成后创建一个博客的文件夹
cd进入博客文件夹

hexo初始化后会在目录里填充一些文件

文件名 作用
_config.yml hexo配置文件
themes 主题文件夹
source 文章源文件
scaffolds 文章模板
public 生成出来的发布文件
node_modules node附属模块

修改hexo配置文件结尾

deploy:
  type: git #类型
  repository: git@github.com:239Pu/239Pu.github.io.git #github项目地址
  branch: master #设置主分支

还可以修改网站资料设置 !!!记得配置文件:后带一个空格 例如 language: zh-CN

如果hexo d命令执行错误可以查看配置文件或ssh连接

安装主题

github下载主题包 放入themes文件夹
修改hexo配置文件

theme: hexo-theme-next #后面跟主题文件夹名`

然后可以进入主题的目录查看主题配置文件是否需要修改
修改完成后 hexo g -d重新生成和上传

常用命令

命令 作用
hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本
命令 缩写
hexo n hexo new
hexo g hexo generate
hexo s hexo server
hexo d hexo deploy
组合命令 作用
hexo s -g #生成并本地预览
hexo d -g #生成并上传
